# XASP 1.6.0 — Empirical First-Passage Discovery

## Delivered

- Empirical XRP +10% / -10% first-passage analysis through 14 days.
- Hourly sampled anchors with minute-precise touch times.
- Conditional mean, restricted mean, median, mode, standard deviation, P25/P75/P90/P95.
- Event rates, first-direction counts, ambiguous same-minute outcomes, and independent clusters.
- Historical max/min excursion Q05/Q50/Q95 for every discovery horizon.
- Minute-return skewness, excess kurtosis, Gaussian tail comparison, and robust outlier counts.
- Volatility-regime breakdowns.
- Human-review candidate-window selection; no automatic model reconfiguration.
- Latest observed completed XRPUSDT candle endpoint so the UI shows price without a trained model.
- UI evidence cards show historical numbers while models remain WAIT; descriptive evidence is clearly separated from live predictions.

## Explicitly not fabricated

The following point-in-time external context series are not yet collected and therefore do not enter training:

- total crypto market capitalization and return;
- XRP market capitalization and market-cap share;
- XRP global volume share;
- XRP volume-to-market-cap turnover.

They remain marked `NOT_COLLECTED` until a versioned point-in-time history source and leakage-safe ingestion pipeline are implemented.

## Governance

- No automated trading.
- No model is promoted because descriptive discovery looks favorable.
- Any horizon revision requires new purged walk-forward training and untouched temporal testing.
- The official action remains WAIT unless the independent empirical gates pass.

Python 3.11 and 3.12 both passed compile, Ruff, strict mypy, pytest, and browser JavaScript syntax checks before release cleanup.
